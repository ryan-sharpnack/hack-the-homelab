# Day 4 — Phase 2: Foundation Build
**Date:** July 9, 2026

## Goals for today
- Build DC01-IRONBRIDGE (Windows Server 2025) → promote as forest root, IRONBRIDGE.LOCAL
- If time and energy allow, begin DC01-HAWTHORNE

## What got done
1. **VM creation.** Built DC01-IRONBRIDGE in Virtual Machine Manager: 2 vCPU, 4096 MB RAM, 60GB qcow2 (SATA bus), UEFI firmware (OVMF 4M.fd, no Secure Boot), Q35 chipset, NIC attached to `vlan10-corp`. Considered VirtIO disk bus for performance but reverted to SATA after weighing the driver-injection overhead against tonight's actual goal — flagged as a deliberate future exercise rather than a rushed addition.
2. **OS install.** Installed Windows Server 2025 (evaluation).
3. **Host identity.** Renamed the machine to `DC01-IRONBRIDGE`, rebooted, confirmed the new name took effect.
4. **Static addressing.** Assigned 10.50.10.10/24 per the IP Addressing Scheme doc. Confirmed the gateway (10.50.10.254) against the live pfSense FW01 console output rather than assuming — verified LAN/OPT1/OPT2 all matched the documented `.254` convention. Set DNS to 127.0.0.1. Verified applied config via `ipconfig /all`.
5. **AD DS role install.** Added the Active Directory Domain Services role via Server Manager, with Group Policy Management and RSAT tools pulled in automatically. Verified destination server as DC01-IRONBRIDGE before installing.
6. **Forest promotion.** Promoted to a new forest, IRONBRIDGE.LOCAL, via the AD DS Configuration Wizard — DNS Server and Global Catalog enabled, DSRM password set, default NetBIOS name and paths, no DNS delegation (expected for a standalone root forest). Server auto-rebooted on completion.
7. **Post-promotion verification.** Confirmed the domain is genuinely healthy, not just visually promoted:
   - `Get-ADDomain` — DNSRoot IRONBRIDGE.LOCAL, DomainMode Windows2025Domain, PDCEmulator DC01-IRONBRIDGE.IRONBRIDGE.LOCAL
   - `Get-DnsServerZone` — AD-integrated primary zones present for IRONBRIDGE.LOCAL and _msdcs.IRONBRIDGE.LOCAL
   - `net share` — SYSVOL and NETLOGON shares present and correctly pathed

## Also today
Investigated a single Error-level DNS Server event (4014) logged shortly after the promotion reboot — "unable to initialize Active Directory security interfaces." Rather than assume a misconfiguration, checked timestamps against surrounding log entries (a Warning at the same second, normal Information activity resuming 16 seconds later), then verified current state directly with `nslookup ironbridge.local` and a re-run of `Get-DnsServerZone`. Both confirmed DNS resolving correctly. Concluded it was a one-time NTDS/DNS boot-order race on first-DC promotion, not a real fault. Full pattern now documented in `IHA-RB-WIN-DC-FOREST-PROMOTION.md` under Known Non-Issues for future DC builds.
